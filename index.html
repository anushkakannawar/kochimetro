AI Model Architecture

Demand Prediction Model

Our passenger demand prediction system uses a stacked LSTM (Long Short-Term Memory) architecture with attention mechanisms:

Model Architecture:
├── Input Layer (features: 48)
│   └── Time, Day, Weather, Events, Historical Data
├── LSTM Layer 1 (128 units, return_sequences=True)
├── Dropout (0.3)
├── LSTM Layer 2 (64 units)
├── Attention Layer
├── Dense (32, activation='relu')
├── Dense (1, activation='linear')
└── Output: Predicted Passenger Count

Training Metrics:
- MAE: 42.3 passengers
- RMSE: 67.8 passengers
- R² Score: 0.947
- Training Time: 2.3 hours

Dynamic Scheduling Algorithm

The AI scheduler uses reinforcement learning (Q-Learning) to optimize train frequency:

1
State Observation
Current passenger load, time of day, historical patterns
2
Action Selection
Adjust frequency (increase/decrease/maintain)
3
Reward Calculation
Minimize wait time + energy cost + crowding penalty
4
Policy Update
Q-table update using Bellman equation
